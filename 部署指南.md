# 企业仓库管理系统 - 部署指南

## 📋 项目简介

企业仓库管理系统是一套前后端分离的 Web 应用系统，采用 Vue 3 + Spring Boot 技术栈开发，用于管理企业的仓库商品、库存、入库、出库等业务。

## 🛠 环境要求

在开始部署之前，请确保您的机器已安装以下软件：

| 软件 | 版本要求 | 说明 |
|:---|:---|:---|
| JDK | 21+ | 后端运行环境 |
| Node.js | 18+ | 前端构建工具 |
| MySQL | 8.0+ | 数据库 |
| Maven | 3.6+ | 后端项目构建工具 |

### 检查环境

```bash
# 检查 Java 版本
java -version

# 检查 Node.js 版本
node -v

# 检查 Maven 版本
mvn -v

# 检查 MySQL 版本
mysql --version
```

## 📦 数据库配置

### 1. 创建数据库

使用 Navicat 或 MySQL 命令行执行：

```sql
CREATE DATABASE warehouse_db 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
```

### 2. 修改数据库连接配置

打开 `backend/src/main/resources/application.yml`，修改以下配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/warehouse_db?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: 你的数据库密码  # 修改为你的 MySQL 密码
```

## 🚀 启动后端服务

### 方式一：使用 Maven 启动（推荐）

```bash
cd backend
mvn spring-boot:run
```

### 方式二：打包后运行

```bash
cd backend
mvn clean package
java -jar target/warehouse-management-1.0.0.jar
```

后端启动成功后，会显示：
```
Started WarehouseApplication in x.xxx seconds
Tomcat started on port 8080
```

## 🎨 启动前端服务

前端启动成功后，会显示：

## 📊 初始化测试数据

### 自动初始化

后端服务启动时，会自动创建初始用户账号。

### 手动初始化数据

如果需要初始化详细的测试数据，请按以下步骤操作：

1. 确保后端服务正在运行（端口 8080）
2. 打开浏览器访问初始化接口：
   - 重置数据: http://localhost:8080/api/data/reset
   - 初始化数据: http://localhost:8080/api/data/init

3. 或者在浏览器控制台执行以下命令：

```javascript
// 重置数据
fetch('http://localhost:8080/api/data/reset')
  .then(res => res.json())
  .then(data => console.log(data));

// 初始化新数据
fetch('http://localhost:8080/api/data/init')
  .then(res => res.json())
  .then(data => console.log(data));
```

## ❓ 常见问题

### 1. 端口被占用

如果启动时提示端口被占用，可以修改端口配置：

**后端端口修改**：修改 `backend/src/main/resources/application.yml`：
```yaml
server:
  port: 8081  # 修改为其他端口
```

**前端端口修改**：修改 `frontend/vite.config.js`：
```javascript
server: {
  port: 5174  // 修改为其他端口
}
```

### 2. 数据库连接失败

检查以下几点：
- MySQL 服务是否启动
- 数据库 `warehouse_db` 是否创建
- `application.yml` 中的用户名密码是否正确
- MySQL 是否允许 root 用户远程连接

### 3. 前端无法访问后端 API

检查后端服务是否正常启动，确保 CORS 配置正确（已在代码中配置）。

### 4. Maven 依赖下载失败

可以配置国内镜像加速下载。修改 `backend/pom.xml` 或 `~/.m2/settings.xml`：

```xml
<mirrors>
  <mirror>
    <id>aliyun</id>
    <mirrorOf>*</mirrorOf>
    <url>https://maven.aliyun.com/repository/public</url>
  </mirror>
</mirrors>
```

### 5. npm 安装依赖慢

可以使用淘宝镜像：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```
