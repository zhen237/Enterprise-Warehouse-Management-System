# 企业仓库管理系统 - 系统设计报告

## 1. 需求分析

### 1.1 项目背景

随着企业规模的不断扩大，仓库管理的复杂度日益增加。传统的手工管理方式已无法满足现代企业对仓库管理的高效、准确要求。本系统旨在通过信息化手段，实现仓库管理的数字化、自动化，提高仓库运营效率，降低管理成本。

### 1.2 功能需求分析

| 序号 | 功能模块 | 功能描述 | 需求来源 |
| :--- | :--- | :--- | :--- |
| 1 | 用户管理 | 用户登录、角色权限管理 | 系统安全性要求 |
| 2 | 商品管理 | 商品信息的增删改查、分类管理 | 基础数据管理 |
| 3 | 仓库管理 | 仓库信息的增删改查，显示商品种类数和库存总价值 | 基础数据管理 |
| 4 | 供应商管理 | 供应商信息管理 | 采购业务支持 |
| 5 | 入库管理 | 入库申请、入库确认、库存更新 | 核心业务流程 |
| 6 | 出库管理 | 出库申请、出库确认、库存更新 | 核心业务流程 |
| 7 | 库存查询 | 实时库存查询、库存预警（显示当前库存、单价、库存价值、最低/最高库存） | 日常运营需求 |
| 8 | 库存盘点 | 盘点记录、差异处理 | 定期盘点需求 |
| 9 | 报表统计 | 库存报表、出入库统计 | 数据分析需求 |

### 1.3 非功能需求

- **性能需求**：系统响应时间不超过2秒
- **安全性需求**：用户密码加密存储，操作日志记录
- **可扩展性**：系统设计支持后续功能扩展
- **易用性**：界面友好，操作简单直观

## 2. 系统设计

### 2.1 架构设计

#### 2.1.1 系统架构

本系统采用前后端分离架构：

- **前端层**：Vue3 + Element Plus，负责用户界面展示和交互
- **后端层**：Spring Boot，提供RESTful API服务
- **数据层**：MySQL数据库，存储业务数据

#### 2.1.2 模块划分

| 模块 | 职责说明 |
| :--- | :--- |
| auth | 用户认证与授权 |
| product | 商品管理 |
| warehouse | 仓库管理（含统计信息） |
| supplier | 供应商管理 |
| inventory | 库存管理（入库、出库、盘点） |
| report | 报表统计 |

#### 2.1.3 核心业务流程图

**入库流程**：
```
用户发起入库申请 → 填写商品/仓库/数量 → 提交申请 → 管理员确认 → 更新库存
```

**出库流程**：
```
用户发起出库申请 → 填写商品/仓库/数量 → 检查库存 → 提交申请 → 管理员确认 → 更新库存
```

**库存预警流程**：
```
实时监控库存 → 库存低于最低库存 → 系统红色警告提示 → 触发补货提醒
```

### 3.4 核心功能实现

#### 3.4.1 仓库统计功能

**后端实现**：
- 新增 `WarehouseStatistics` DTO，包含仓库基本信息和统计数据
- 在 `InventoryRepository` 中添加库存价值统计查询方法
- 在 `WarehouseService` 和 `WarehouseServiceImpl` 中实现统计逻辑
- 新增 `/api/warehouses/statistics` API 接口

**前端实现**：
- 修改 `Warehouses.vue`，调用统计API
- 显示商品种类数和库存总价值
- 使用表格汇总功能

#### 3.4.2 库存查询功能

**后端实现**：
- `InventoryRepository` 支持按仓库筛选库存
- 库存实体关联商品和仓库信息

**前端实现**：
- 修改 `Inventory.vue`，显示当前库存、单价、库存价值
- 添加最低库存和最高库存列
- 库存低于最低库存时红色警告提示
- 使用表格汇总功能

#### 3.4.3 库存预警机制

- **最低库存**：作为安全库存，库存低于此值时显示红色警告
- **最高库存**：作为库存上限，控制库存成本
- 预警颜色标识：红色表示库存不足，黑色表示正常

## 6. 代码安全性

### 6.1 注意事项

1. **密码加密**：使用BCryptPasswordEncoder加密存储用户密码
2. **SQL注入防护**：使用JPA预编译语句，避免SQL注入
3. **跨域配置**：配置合理的CORS策略
4. **异常处理**：统一异常处理，避免敏感信息泄露
5. **输入验证**：对用户输入进行验证和过滤
6. **权限控制**：基于角色的访问控制

### 6.2 已实现的安全措施

| 安全措施 | 实现方式 |
| :--- | :--- |
| 密码加密 | BCryptPasswordEncoder |
| SQL注入防护 | Spring Data JPA |
| CORS配置 | WebConfig配置允许前端访问 |
| 全局异常处理 | GlobalExceptionHandler |
| JWT认证 | Spring Security + JWT Token |
| 禁用CSRF | RESTful API场景禁用CSRF |
